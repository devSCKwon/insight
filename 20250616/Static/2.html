그룹웨어 문서 공유 및 보안 문서 관리 가이드

기업 내 문서 열람 권한 판단 및 안전한 관리 방안

개요

기업 내 그룹웨어 시스템을 통해 다양한 문서가 작성·결재·공유되는 가운데, 민감도와 활용 목적에 따라 문서의 열람 권한과 공유 방식이 달라질 수 있습니다. 본 문서는 사용자가 자율적으로 문서 열람 권한을 판단하고, 비밀등급 문서를 안전하게 관리하며, 팀 자산에 적합한 서식 환경을 설정할 수 있도록 가이드를 제공합니다.

1. 문서 열람 판단 방법 (작성자 기준)

작성자가 문서 조회 가능 여부를 판단할 수 있는 네 가지 방식과 대표 예시는 다음과 같습니다.

방법 설명 예시 특이사항
① 결재라인 참조(자동동의) 기안 시 참조자로 지정된 사용자는 결재선에 포함되지 않아도 열람 가능 - 그룹웨어 극비/대외비 샘플 서식
- 영업비밀 등급 자체확인 서식		 문서 열람 시 별도 승인 절차 없음
② 열람문서 기능 서식 내 열람 권한 내장. 결재라인 외에도 열람 가능 - 지출품의서 (재경부 확인 목적) 결재문서 리포트 통계 활용 시 열람 권한 필수
③ 문서공유 특정 부서 또는 사용자에게 문서 직접 공유 - 작성부서의 정책안내문을 타부서에 공유 실시간 협업에 적합, 공유 시점 및 범위 통제 가능
④ 수신함 지정 결재선과 무관하게 업무 수행자(팀 또는 개인) 지정 가능 - 협조전 수신 → 재기안 필요
- 팀 단위 수신 시 담당자 지정		 수신자별 완료 이력 기록 가능

2. 향후 개선 계획

2.1 공용서식 내 비밀등급 부여 기능 고도화

목표: 비밀 문서의 무분별한 확산 방지 및 결재권자 중심의 문서 보안 강화

  • 비밀등급 필드 활성화: 공용서식에 비밀등급 필드 활성화.
  • 결재권자 최종 판단: 결재권자가 문서 민감도를 최종 판단하고 비밀등급 지정.
  • 공유 기능 차단: 비밀등급 문서는 공유 기능 자동 차단, 참조(자동동의)로만 열람 가능.
  • 별도 서식 지정: 일반/비문 혼재 가능성 있는 경우, 별도 서식 지정.
  • 예시: 결재자가 ‘비문’으로 지정한 기안서식은 공유 버튼 비활성화 + 참조자만 열람 가능.

2.2 팀정보자산관리자의 별도 서식 생성

목표: 팀 단위 보안자산의 고유 특성에 맞는 문서관리 환경 구축

  • 팀별 전용 서식 생성: 팀별 전용 서식을 생성하여 서식 수준에서 공유 범위 제어.
  • 공유 대상 임의 변경 불가: 문서 작성자는 공유 대상자를 임의로 변경할 수 없음.
  • 열람 가능 대상 고정: 민감한 업무는 서식 생성 단계에서 열람 가능 대상만 고정 지정.
  • 예시: 원가영업팀의 ‘OO기안문’은 원가영업팀장만 공유 가능하도록 서식 설정 (타부서 사용자 작성 불가, 공유 범위 변경 불가).

3. 기타 고려 사항

  • 그룹웨어 게시판에 "결재선 이해하기" 가이드 게시.
  • 문서 열람/공유 이력 리포트 정기 생성 및 관리자 모니터링.
  • 서식 유형별 열람·공유 권한 체계 수립 (일반/대외비/비문 등).

결론

본 가이드는 그룹웨어 내 문서 열람과 보안 문서 관리에 대한 실무 적용 기준을 정립하고자 작성되었습니다. 향후 기능 개선과 함께 사용자, 부서, 결재권자 간 명확한 역할과 책임 분담이 이루어질 수 있도록 지속적인 개선이 필요합니다.
